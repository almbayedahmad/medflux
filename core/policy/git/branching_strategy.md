# Branching Strategy

Branch naming and flow for development, releases, and hotfixes.

- Main Branches
  - main: production code
  - develop (optional): integration branch

- Supporting Branches
  - feature/<topic>
  - bugfix/<topic>
  - hotfix/vX.Y.Z
  - release/vX.Y.Z

- Tags
  - vX.Y.Z (SemVer); sign when possible

- Merging
  - PRs with required reviews; rebase or squash per repo setting
